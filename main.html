üé® Design Guidelines & Specifications

Typography & Readability

  • Primary Font: Segoe UI, 18px+ minimum for body text
  • Button Text: 20-24px, bold weight for emphasis
  • Headlines: 28-32px, high contrast colors
  • Line Height: 1.6 for optimal reading comfort

Color Palette & Contrast

  • Primary Blue: #3498db (buttons, accents)
  • Success Green: #2ecc71 (healthy status)
  • Warning Orange: #f39c12 (attention needed)
  • Alert Red: #e74c3c (requires immediate care)
  • Background: #f8f9fa (low strain, high readability)

Button Design & Spacing

  • Minimum Size: 44px height (Apple HIG standard)
  • Touch Target: 48px+ with adequate spacing
  • Border Radius: 15-20px for modern, friendly appearance
  • Margin: 20px between major elements

Accessibility Features

  • Icons: Paired with text labels, emoji-based for universality
  • Visual Feedback: Color + icon + text for status communication
  • Error Prevention: Clear guidance and confirmation dialogs
  • Navigation: Simple back buttons, consistent placement

Photo Capture UX

  • Visual Guide: Dashed border overlay for teeth positioning
  • Instructions: Step-by-step text above camera view
  • Capture Button: Large, red, circular - intuitive camera metaphor
  • Feedback: Immediate visual confirmation after capture

Results Presentation

  • Three-Tier System: Healthy (green), Attention (orange), Risk (red)
  • Visual Hierarchy: Icon ‚Üí Status ‚Üí Description ‚Üí Actions
  • Language: Simple, non-medical terms for clarity
  • Action Buttons: Clear next steps, max 2 options to avoid confusion
